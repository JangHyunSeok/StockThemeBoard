# 프로젝트 구조 가이드

## 디렉토리 구조

본 프로젝트는 다음과 같은 디렉토리 구조를 따릅니다:

```
StockThemeBoard/
├── doc/                    # 프로젝트 설계/개발 및 지침 문서
├── nginx/                  # Nginx 설정 및 SSL Proxy
├── certbot/                # Let's Encrypt 인증서 및 ACME challenge
└── src/                    # 소스 코드
    ├── frontend/          # 프론트엔드 소스
    └── backend/           # 백엔드 소스
```

## 소스 코드 배치 규칙

### 프론트엔드 소스

- **위치**: `src/frontend/`
- **용도**: 사용자 인터페이스, 클라이언트 사이드 로직
- **예시**: React, Vue, Angular 등의 프레임워크 코드, HTML, CSS, JavaScript 파일

### 백엔드 소스

- **위치**: `src/backend/`
- **용도**: 서버 사이드 로직, API, 데이터베이스 연동
- **예시**: Node.js, Python, Java 등의 서버 코드, API 엔드포인트, 비즈니스 로직

## 주의사항

> [!IMPORTANT]
> 앞으로 모든 개발 작업은 위의 디렉토리 구조를 준수하여 진행합니다.

- 프론트엔드 관련 코드는 반드시 `src/frontend/` 하위에 배치
- 백엔드 관련 코드는 반드시 `src/backend/` 하위에 배치
- 새로운 기능 추가 시에도 이 구조를 유지
- 공유되는 타입이나 유틸리티가 필요한 경우 `src/shared/` 디렉토리를 고려할 수 있음

## 문서 구조

프로젝트 문서는 `doc/` 디렉토리에 단계별로 정리합니다:

- **01. 분석**: 요구사항 분석, 분석 문서
- **02. 설계**: 설계 문서, 아키텍처 다이어그램
- **03. 개발**: 개발 가이드, 구조 문서, TODO 리스트
