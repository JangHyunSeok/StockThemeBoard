## 🔄 주요 변경사항 (v1.5.1)

### 1. KIS 토큰 중복 호출 제거
- **문제**: `volume-rank-by-theme` API 1회 요청 시 `get_access_token()` 조용이 최대 32회(일반 요청 2회 + 섹터 조회 30회) 호출되던 구조적 문제
- **해결**: 엔드포인트 진입 시 `access_token = await kis_client.get_access_token()` 1회만 호출 후, `get_volume_rank` / `get_stock_quote` / `get_index_quote` 전체에 공유 토큰을 주입하도록 리팩토링
- **효과**: API 요청 1회당 Redis 조회 32회 → 1회로 삭감

### 2. `get_stock_quote`, `get_volume_rank`, `get_index_quote` 시그니처 변경
- 각 메서드에 `access_token: str = None` 파라미터 추가
- 미전달 시 내부 fallback으로 자동 발급 (하위 호환 유지)
- 호출자단에서 토큰 상단 주입 시 Redis 쿄리 경유없이 바로 사용 가능

### 3. `indices.py` 30초 Redis 캐시 추가
- 지수 폴링(전체 30초 갱신) 시마다 KIS API를 직접 호출하던 문제 해결
- `indices:current` Redis 키로 30초 캐싱 도입, 폴링 시 Redis HIT 시 KIS API 미호출
- 코스피 / 코스닥 조회뗄 토큰 1회 선발급 후 2회 조회에 공유 적용

### 4. `jobs.py` 스케줄러 중복 틀 제거
- `fetch_and_save_krx_rankings()`, `fetch_and_save_nxt_rankings()` 내 불필요한 `await kis_client.get_access_token()` 2군 삭제
- `get_volume_rank()` 내부 fallback으로 대체
