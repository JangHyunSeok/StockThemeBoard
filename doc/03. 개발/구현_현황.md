## 🔄 주요 변경사항 (v1.5.0)

### 1. KIS API 통신 안정성 극대화 (Pre-warming)
- 매일 07:50 장 시작 전, 스케줄러를 통해 KIS API 토큰(수명 24시간)을 무조건 새로 발급받도록 캐시 로직 고도화.
- 장중(09:00 ~ 15:30)에 사용자 요청 시점과 토큰 만료 시점이 겹쳐 발생하던 통신 지연 현상 완벽 방어.

### 2. Timezone 기반 저장 정확도 확보
- 비동기 DB 처리중의 timezone 충돌 처리. Python 코드단에서 명시적으로 `datetime.now()`를 통한 KST 시각 `created_at` 부여.
- 스케줄러 놓침 발생 시 즉시 서버 구동 시간 기록.
